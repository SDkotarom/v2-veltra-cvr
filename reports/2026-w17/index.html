01Conclusion — 結論
W17 / サイト障害インシデント（4/24〜4/26朝）
W17 後半に AWS Memcached ノード障害でサイト全体が 504 Gateway Time-out。
4/24（金）CVR 0.66% / 4/25（土）0.96% と通常の 1/2〜1/3 まで激減し、推定 −1,450件 / −¥9.4M の取りこぼし。
4/26（日・W18初日）の CVR 1.22% への即時回復が、これが 「障害による一過性の落ち込み」であることを裏付ける。
ユーザー行動の構造的悪化ではない — 障害がなければ W17 は前週並みの水準で着地していた。
W17 の見かけ CVR 1.10% は、3 つの独立した特殊要因が重なった結果: (1) 4/24〜4/25 のサイト障害（504 で「フォーム入っても完了できない」状況、最大の負の要因）、 (2) Singapore × Chrome × Win bot の継続（W16 から半減も指紋一致で残存、分母希釈）、 (3) 4/20 為替マージン変更（EU 3.9% / TWD・SGD・MYR 0% / 他 2.9%、影響は小さく中立〜微プラスと推定）。
今週優先するべきは、障害の再発防止アラート整備と、bot WAF ブロック。 ユーザー体験施策（BN1〜BN3）は GW 繁忙期前にこの 2 件と並行で着手する。

E.04 — 4/20 為替変更（中立〜微プラス想定）
EU 3.9% / TWD・SGD・MYR 0% / その他 2.9% — 主に値下げ方向で CVR には不利には働きにくい
~0
価格戦略チームによる 4/20 適用、CVR 変動への寄与は今週の数字には小さい想定
為替マージン引き下げは 表示価格を下げる方向に働くため、CVR には基本プラス寄与（特に Singapore・台湾・マレーシアからの購買者）。 kotarom（CVR チーム）の Slack でも「普通そちらは良い方向に働きそう」と言及。W17 の見かけ CVR 悪化を為替で説明することはできない。 来週 W18 のレポートで通貨×エリア別の CVR/購入数を切り出して、為替変更の純効果を分離測定する（障害と bot のノイズが除けるため）。
03Action — やるべきこと
障害周知フローの整備 + WAF ブロック完了 + GW 前に BN1〜BN3 着手の 3 本立て
障害・bot・構造課題の 3 つを並行で潰す。意思決定を誤らないため、見かけ CVR ではなく「障害除外 + bot 除外」の実質値で議論する習慣を本格適用。
1
NEW / 緊急 障害対応 / 周知フロー 優先度: 最高
サイト障害の社内周知フローを整備（4/24〜4/25 で広告チームへの通知が遅れた）
検知自体は SRE 側で行われていたが、SEM 広告チームが「CVR 急落の原因が障害だ」と把握したのは 4/27（月）の朝。その間、広告投資の判断や CVR 報告に「障害情報なし」の状態でデータを見ていた。 同じ事象を二度起こさないため、障害発生時に #urgent-issues から自動で関係チャネル（CVR / SEM / マーケ / 経営）へ周知される運用を Mayumi さん・Jeff さんと合意する。 また、障害日が含まれる週のレポートには「障害アノテーション」を必ず記載し、見かけの数字に引きずられないようにする。
ステップ: ① #urgent-issues から関係チャネルへの自動転送設定（1日）② annotations.json に障害イベントを記録するルール文書化（半日）③ CVR ダッシュ KPI に「障害日除外」トグルを追加（中期）
2
引き継ぎ / 緊急 WAF / インフラ 優先度: 最高
WAF ブロックを今週中に完了 — bot は週次バッチ化して継続中
W16 の Action 1 で書いた AWS ap-southeast-1 / Singapore × Chrome × Windows ブロックが 未だ未実装のまま、bot は週次バッチに切り替えて継続発信中（4/20・4/21・4/24 にスパイク）。 このまま GW（5/1〜5/6）を迎えると 繁忙期で再びピークを打つリスクがある（W16 ピーク 4/15 の規模なら raw CVR が再び 1% を割る）。 AWS IP レンジ JSON 投入 + Singapore × Chrome × Windows × Referer 空 fingerprint ルールの 2 段構えを 今週中（4/27〜5/2）に必達。
3
既存 BN Top3 UX / フォーム / AC 優先度: 高
GW 前に「決済直前の諦め」「AC に辿り着けないモバイル/リピーター」を着手
Top3 ボトルネックは W16 と同じ顔ぶれが 2 週連続。障害・bot 騒動が一段落するまで先送りしていると、GW 繁忙期に「ボトルネックは知っていたが手は打てていない」状態になる。 BN1〜BN3 のうち quick wins（effort=S, 制約なし）から先行着手すれば 1〜2週間で実装可能。
BN1: 新規ユーザーが『長いフォーム＋最終価格の不意打ち』で決済直前に諦めている（新規 ④→⑤ -26.7pp / 月60万人影響）
BN2: モバイルで『自分に合うAC』を見つけられず、比較疲れで離脱している（モバイル ①→② -24.1pp / 月50万人影響）
BN3: リピーターが『前回見たAC』を思い出せず、探し疲れで離脱している（リピーター ①→② -28.2pp / 月34万人影響）
4
継続 / 計測基盤 ダッシュボード / 為替分析 優先度: 中
CVR は「raw / 障害除外 / bot 除外」の 3 本立てを kpi.html に反映 + 為替変更の通貨別効果測定
W16 で書いた raw / bot 除外の 2 本立てに、「障害日除外」軸を追加。意思決定の精度を上げる。 また、4/20 為替変更の純効果を W18 のレポートで通貨×エリア別 CVR で切り出し測定する（W17 は障害と bot で交絡しているため、W18 の純粋な比較が必要）。
