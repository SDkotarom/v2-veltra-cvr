特定結果
Organic Search × リピーター の ③→④ 意向転換率が 63.86%（全体平均 86.62% の -26.3%、-22.76pp）。
月間 451,499セッション。①→② 36.29%・②→③ 89.63%と上流ファネルは良好なため、カレンダー閲覧後の「意向決定」フェーズ特有の課題が存在する。価格変動への不安やリピーター向けUI不足が主因として疑われる。
原因仮説 × 3
なぜ Organic Search × リピーターの ③→④ 意向転換率が低いか？
仮説 1（有力）
価格変動への戸惑いで離脱
リピーターがカレンダーで日程・価格を確認した際、前回予約時から料金が変わっている（繁忙期値上がり等）ことに戸惑い意向決定できない。Organic リピーターは直接アクセスではなく検索経由で来るため、目的商品を再検索して他サービスと比較している可能性もある。
裏付けデータ:
  • ②→③ 89.63%（良好）なのに③→④が63.86%に急落 → カレンダー閲覧後の価格確認で離脱
  • Organic 検索経由 = 比較検討意識が高いユーザー層
仮説 2
前回体験への不満で再予約モチベーションが低い
前回の予約体験（ツアー品質・対応など）に不満があり再予約の動機が弱い。カレンダーまで到達しても決断できずに離脱する。ただしCVR 1.50%がベースラインと同程度であることから、問題は一部のユーザーに限定される可能性が高い。
裏付けデータ:
  • CVR 1.4975% はベースラインと同程度 → 全リピーターの問題ではなく一部の離脱
  • Organic 検索での再訪 = 特定商品への直接意図ではなくブラウジングの可能性
仮説 3
リピーター向け再予約UIの欠如
ACページにリピーター向けの「また予約する」優遇UI（前回予約情報の自動入力・前回プランへのショートカット）がなく、一から手動入力する手間が意向決定を阻む。前回と同じプランを探す操作が煩雑で離脱を招いている。
裏付けデータ:
  • ①→②・②→③が良好 → リピーターは商品認識・検討は問題なく、意向決定ステップ固有の障壁がある
  • Direct × リピーターとの③→④比較で差があれば、Organic 経由特有の問題を絞り込める
打ち手 × 3
Organic Search × リピーターの ③→④ 意向転換率改善施策
施策 A
③→④ +5pt
「前回の予約」バッジ + 同プラン再予約CTA
リピーターがACページを訪問した際、前回予約した同一または類似商品であれば「前回ご予約いただきました」バッジを表示し、前回プランへのクイックアクセスCTAを追加。再予約の摩擦を最小化する。
開発仕様 ・予約履歴 API で該当商品の過去予約を照合（ログイン状態必須）
・ACページ上部に「前回 2025年○月ご予約 → 同じプランで再予約」CTAバナー配置
・前回選択プラン・人数を自動セット、カレンダーへダイレクト遷移
・未ログイン時: ログイン促進バナーを表示
施策 B
③→④ +3pt
価格変動の透明性確保（前回比表示）
カレンダーの価格表示に「前回予約時より +○円」「前回予約時より -○円」などの変動表示を追加。価格変化への戸惑いを除去し、納得した上で意向決定できるよう透明性を高める。
開発仕様 ・予約履歴から前回支払い価格を取得し、現在価格と差分を計算
・カレンダー上の日程・価格セルに差分バッジを追加表示（+赤 / -緑）
・「なぜ価格が変わったか」ツールチップ（繁忙期・残席数等）
・プライバシー: 予約履歴はログインユーザーのみ参照
施策 C
③→④ +4pt
リピーター向けポイント還元インセンティブの表示
ACページおよびカレンダー閲覧時に、リピーター向けポイント還元率の優遇情報（「今回予約で通常の2倍ポイント」等）を目立つ形で表示。再予約のインセンティブを強化し意向決定を後押しする。
開発仕様 ・リピーター判定: ログイン + 過去予約1件以上
・ACページのCTAエリアにポイント還元バッジを配置
・「予約完了でXXXポイント（通常の2倍）」動的計算表示
・ポイントプログラムがない場合: クーポン施策として代替実施
検証方法
ABテスト 2週間。対象: Organic Search × リピーターユーザー（ログイン済み）。
主要指標: ③→④ 意向転換率（現状 63.86% → 目標 70%以上）。
副次指標: CVR変動、ポイント利用率、リピート予約率（30日以内）。
施策Aから優先実施。ログイン率が低い場合は施策Cのクーポン版を非ログインにも展開。
チームレビュー欄
施策実施前にチームで確認してください。
