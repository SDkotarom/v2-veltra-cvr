# VELTRAサイト 現仕様スナップショット

> **目的**: 打ち手提案時に「既にある機能を提案してしまう」事故を防ぐため、ページ単位で"今サイトに何があるか"を固定化するドキュメント。
>
> **更新タイミング**: ユーザーから口頭で新機能の存在を教わった都度／サイト改修があった都度。
>
> **関連ファイル**:
> - `known-implementations.json`: キーワード照合用カタログ（機械チェック）
> - `annotations.json`: リリースイベントログ（いつ・誰が・何を出したか）
> - `docs/captures/`: PC/SPスクリーンショット（視覚的裏付け）
>
> **使い方（Claude向け）**: Phase 2 のボトルネック分析に入る前にこのファイルを**必ず読む**。打ち手を考える段階で該当ページのセクションを再確認し、既存機能と重複していないかチェック。ユーザーから「覚えて: XX」と言われたら該当ページセクションに追記 + `known-implementations.json` にも1行追加。
>
> **最終更新**: 2026-04-20（初版スケルトン作成）

## 追記テンプレート（「覚えて」用）

ユーザーから機能の存在を教わった時、以下フォーマットでClaude側がEditする:

```markdown
### 実装済み機能
- **{機能名}**（{YYYY-MM}、{PC/SP/両}）: {1行説明}。{ユーザーから教わった日: YYYY-MM-DD}
```

そして `known-implementations.json` の該当カテゴリに:

```json
{
  "title": "{機能名}",
  "date": "YYYY-MM",
  "keywords": ["キーワード1", "キーワード2"],
  "detail": "{1行説明}"
}
```
